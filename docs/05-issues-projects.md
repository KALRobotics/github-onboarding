# Issues and Projects

Issues = task tracking for your team.

## Create Issue
1. Repo → Issues → New issue
2. Clear title
3. Details and acceptance criteria
4. Assign and label

## Working on Issue
1. Assign yourself
2. Create branch: `feature/name/issue-123-description`
3. Reference in PR: "Fixes #123"
4. Auto-closes when merged!

## GitHub Projects
Kanban board with columns:
- Backlog → This Week → In Progress → Review → Done

Move your cards as status changes!
